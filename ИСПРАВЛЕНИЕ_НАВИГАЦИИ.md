# Анализ и исправление навигации в Header.tsx

## 🔍 Обнаруженная проблема

В компоненте [`Header.tsx`](components/layout/Header.tsx:14-18) обнаружена критическая ошибка в конфигурации навигации:

```typescript
const navigation = [
  { name: 'Главная', href: '/dashboard', icon: Home },
  { name: 'Курсы', href: '/dashboard', icon: BookOpen },  // ❌ ОШИБКА!
  { name: 'Профиль', href: '/profile', icon: User },
];
```

### Суть проблемы:

1. **Дублирование роута**: Пункты меню "Главная" и "Курсы" указывают на один и тот же маршрут `/dashboard`
2. **Проблема с активным состоянием**: Оба пункта меню одновременно подсвечиваются как активные при проверке `pathname === item.href` (строки 46, 95)
3. **Невозможность перехода**: Клик на "Курсы" не приводит к переходу на страницу каталога курсов
4. **UX проблема**: Пользователь не может попасть в каталог курсов из навигации

## 🎯 Ожидаемое поведение после исправления

### Desktop (≥768px):
1. Пользователь видит три пункта меню
2. При клике на "Главная" → переход на `/dashboard`
3. При клике на "Курсы" → переход на `/courses`
4. При клике на "Профиль" → переход на `/profile`
5. Активный пункт подсвечивается синим фоном и темным текстом

### Mobile (<768px):
1. Пользователь нажимает на иконку меню (гамбургер)
2. Раскрывается мобильное меню с тремя пунктами
3. При клике на любой пункт:
   - Происходит переход на соответствующую страницу
   - Меню автоматически закрывается
4. Активный пункт подсвечивается

### Навигационный flow:
```
Главная (/dashboard)
  ↓
  "Мои курсы"
  - Купленные курсы (с прогрессом)
  - Бесплатные курсы
  - Кнопка "Детали курса"
  - Кнопка "Продолжить/Начать"

Курсы (/courses)
  ↓
  "Все курсы"
  - Каталог всех курсов
  - Карточки с прогрессом
  - Статус (Полный доступ / В процессе / Бесплатные уроки)
  - Клик на карточку → переход на /course/[courseId]

Профиль (/profile)
  ↓
  Информация о пользователе
  - Персональные данные
  - Настройки
```

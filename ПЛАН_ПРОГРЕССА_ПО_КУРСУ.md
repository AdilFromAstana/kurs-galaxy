# План реализации прогресса по курсу с последним уроком

## Проблема
На странице курса не синхронизированы:
1. Прогресс по конкретному курсу (сколько уроков пройдено)
2. Последний просмотренный урок (какой урок смотрел последним в ЭТОМ курсе)
3. Визуальная индикация этого прогресса

## Текущая ситуация

### Что уже есть ✅
- [`lib/storage.ts`](lib/storage.ts) - функции для работы с localStorage
  - `getProgress(courseId)` - получить прогресс по курсу
  - `addToProgress(lessonId, courseId)` - добавить урок к прогрессу
  - `getLastLesson()` - получить последний урок (НО: глобально, без привязки к курсу)
  - `setLastLesson(lessonId)` - сохранить последний урок (НО: глобально)

- [`hooks/useProgress.ts`](hooks/useProgress.ts) - хук для работы с прогрессом
  - Принимает `courseId` и возвращает прогресс по этому курсу
  - `getProgressPercentage()` - процент завершения
  - `getCompletedCount()` - количество завершенных уроков
  - `isLessonCompleted()` - проверка завершения урока

### Что НЕ работает ❌
1. **`getLastLesson()` глобальная** - сохраняет последний урок для ВСЕХ курсов, а не для конкретного
2. **На странице курса** [`app/course/[courseId]/page.tsx`](app/course/[courseId]/page.tsx:75):
   - Использует `getLastLesson()` без привязки к текущему курсу
   - Если последний урок был из другого курса - показывает его неправильно

## Решение

### 1. Изменить структуру хранения последнего урока

**Было:**
```typescript
localStorage: {
  'nail_last_lesson': 'lesson-1-1'  // глобально
}
```

**Станет:**
```typescript
localStorage: {
  'nail_last_lesson': {
    'brow-master-pro': 'lesson-1-1',
    'nail-design-pro': 'lesson-2-3',
    // ... для каждого курса свой последний урок
  }
}
```

### 2. Обновить функции в [`lib/storage.ts`](lib/storage.ts)

**Изменения:**
```typescript
// Старая версия (глобальная)
export const getLastLesson = (): string | null => { ... }
export const setLastLesson = (lessonId: string): void => { ... }

// Новая версия (по курсу)
export const getLastLesson = (courseId?: string): string | null => { ... }
export const setLastLesson = (lessonId: string, courseId: string): void => { ... }

// Внутренний тип
type LastLessonByCourse = Record<string, string>;
```

### 3. Обновить хук [`hooks/useProgress.ts`](hooks/useProgress.ts)

**Изменения:**
```typescript
export const useProgress = (courseId: string = 'brow-master-pro') => {
  // ...
  
  const addLesson = (lessonId: string, cId?: string): void => {
    const targetCourseId = cId || courseId;
    addProgress(lessonId, targetCourseId);
    setProgressState(getProgress(targetCourseId));
    setLastLesson(lessonId, targetCourseId); // ← передаем courseId
  };
  
  // Новый метод
  const getLastLessonId = (cId?: string): string | null => {
    const targetCourseId = cId || courseId;
    return getLastLesson(targetCourseId); // ← передаем courseId
  };
  
  return {
    // ...существующие
    getLastLessonId, // ← новый метод
  };
};
```

### 4. Обновить страницу курса [`app/course/[courseId]/page.tsx`](app/course/[courseId]/page.tsx)

**Изменения:**
```typescript
export default function CoursePage() {

  const {

```

4. Открыть другой курс → должен показаться прогресс по ЭТОМУ курсу

## Преимущества решения

✅ **Изоляция по курсам** - прогресс и последний урок хранятся отдельно для каждого курса

✅ **Обратная совместимость** - миграция старых данных не потеряет прогресс пользователя

✅ **Расширяемость** - легко добавить новые курсы без изменения логики

✅ **Локальное хранение** - все в localStorage, быстро и без сервера

✅ **Типобезопасность** - TypeScript типы для всех структур данных

## Риски и меры

⚠️ **Риск 1:** Потеря данных при миграции
- **Мера:** Тщательная миграция старых данных в новый формат

⚠️ **Риск 2:** Несоответствие courseId между страницами
- **Мера:** Всегда передавать courseId явно, не полагаться на дефолтные значения

⚠️ **Риск 3:** Прогресс не обновляется в реальном времени
- **Мера:** useEffect следит за изменениями в localStorage (уже реализовано)

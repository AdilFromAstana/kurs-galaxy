# План исправления модалки покупки для мобильной версии

## Проблемы на скриншоте

Проанализировав скриншот и код [`PurchaseModal.tsx`](components/modals/PurchaseModal.tsx), выявлены следующие проблемы:

1. **Контент обрезается снизу** - не видно кнопки "Безопасная оплата" и часть нижних элементов
2. **Слишком много контента для мобильного экрана** - модалка не помещается полностью
3. **Отсутствует прокрутка** - хотя в [`globals.css`](app/globals.css:108) есть `max-h-[90vh] overflow-y-auto`, контент все равно обрезается
4. **Неоптимальные отступы** - много вертикального пространства съедается заголовками и отступами

## Диагностика проблем

### Текущая структура модалки:

```
modal-overlay (строка 52)
  └─ modal-content max-w-3xl (строка 53)
       ├─ Close Button
       ├─ Header (иконка + заголовки) - занимает ~180px
       ├─ Pricing Plans (grid) - занимает ~300px 
       ├─ Features (список из 4 пунктов) - занимает ~150px
       ├─ CTA Buttons - занимает ~60px
       └─ Security Info - занимает ~40px
```

**Общая высота:** ~730px  
**Доступная высота на мобилке:** ~600-700px (зависит от устройства)

### Проблемы в коде:

1. **`max-w-3xl` на модалке** (строка 53) - слишком широкая максимальная ширина для мобилки
2. **Фиксированные отступы** - `mb-6` между секциями слишком большие для мобилки
3. **Grid для планов** - на мобилке показывается по одному плану, но занимает много места
4. **Большая иконка и header** - занимают слишком много места наверху

### 6а. Блокировка скролла body при открытой модалке

**Файл:** [`components/modals/PurchaseModal.tsx`](components/modals/PurchaseModal.tsx:26-29)

Добавить `useEffect` для блокировки прокрутки основной страницы:

```tsx
useEffect(() => {
  const loadedCourse = getCourseWithPricing(courseId);
  setCourse(loadedCourse || null);
}, [courseId]);

// Добавить новый useEffect:
useEffect(() => {
  if (isOpen) {
    // Блокируем скролл body
    document.body.style.overflow = 'hidden';
    // Для iOS Safari
    document.body.style.position = 'fixed';
    document.body.style.width = '100%';
  } else {
    // Возвращаем скролл
    document.body.style.overflow = '';
    document.body.style.position = '';
    document.body.style.width = '';
  }

  // Cleanup при размонтировании
  return () => {
    document.body.style.overflow = '';
    document.body.style.position = '';
    document.body.style.width = '';
  };
}, [isOpen]);
```

## Дополнительные улучшения

### 11. Более компактная типографика для мобилки

**Общий подход:** Сделать все тексты меньше и с меньшими межстрочными интервалами на мобилке:

```tsx
// Заголовок модалки
<h2 className="text-xl md:text-2xl lg:text-3xl font-bold text-dark-900 mb-1 md:mb-2">
  Выберите тарифный план
</h2>
<p className="text-sm md:text-base lg:text-lg text-dark-600 leading-snug">
  Получите доступ к курсу на удобный для вас период
</p>

// Названия планов
<h3 className="text-base md:text-lg lg:text-xl font-bold text-dark-900 mb-1 md:mb-2 text-center leading-tight">
  {plan.name}
</h3>

// Цена
<div className="flex items-baseline justify-center gap-1">
  <span className="text-2xl md:text-3xl font-bold text-dark-900">
    {plan.price.toLocaleString()}
  </span>
  <span className="text-base md:text-lg text-dark-600">{plan.currency}</span>
</div>

// Период доступа
<span className="text-xs md:text-sm text-dark-600">
  Доступ: <span className="font-semibold">{plan.accessPeriod.label}</span>
</span>

// Описание плана
{plan.description && (
  <p className="text-xs md:text-sm text-dark-600 text-center leading-snug">
    {plan.description}
  </p>
)}
```
